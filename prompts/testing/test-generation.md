# Prompt mẫu cho Kiểm thử (Testing)

## 1. Tạo Unit Test

```
Hãy viết unit tests cho hàm sau:
[Dán hàm ở đây]

Framework sử dụng: [Jest/Vitest/Mocha]

Yêu cầu các trường hợp test:
- Trường hợp dữ liệu chuẩn (Happy path)
- Các trường hợp biên (Edge cases)
- Các trường hợp gây lỗi (Error scenarios)

Yêu cầu: Sử dụng mô phỏng (Mocks) cho các phụ thuộc bên ngoài nếu có.
```

---

## 2. Tạo Integration Test (Kiểm thử tích hợp)

```
Viết integration tests cho [tên API hoặc Component]:

Các kịch bản cần kiểm tra:
1. [kịch bản 1]
2. [kịch bản 2]

Thiết lập:
- Database: [cách thức chuẩn bị dữ liệu test]
- Mocks: [những gì cần giả lập]
```

---

## 3. Viết E2E Test (Kiểm thử đầu cuối)

```
Viết bài kiểm tra E2E cho quy trình người dùng:
[Mô tả quy trình, ví dụ: Đăng ký -> Xác thực email -> Đăng nhập]

Framework: [Playwright/Cypress]
Yêu cầu: Bao gồm các bước kiểm tra thành công và thông báo lỗi khi nhập sai.
```
