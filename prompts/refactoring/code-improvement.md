# Prompt mẫu để Cải thiện code (Code Improvement)

## 1. Tối ưu hóa Clean Code

```
Hãy refactor đoạn code này để tăng khả năng đọc và bảo trì:

[Dán code ở đây]

Áp dụng các nguyên tắc sau:
- Nguyên tắc đơn nhiệm (Single Responsibility Principle)
- Đặt tên biến/hàm có ý nghĩa
- Giảm độ phức tạp (Complexity)
- Loại bỏ code trùng lặp
- Thêm chú thích cần thiết

Yêu cầu: Giữ nguyên logic ban đầu và giải thích chi tiết các thay đổi bạn thực hiện.
```

---

## 2. Tối ưu hóa Hiệu suất (Performance)

```
Tối ưu hóa đoạn code này để đạt hiệu suất tốt hơn:

[Dán code ở đây]

Vấn đề hiện tại:
- [mô tả vấn đề: tốn RAM, chạy chậm, nhiều vòng lặp...]

Tập trung vào:
- Độ phức tạp thời gian (Time complexity)
- Sử dụng bộ nhớ (Memory usage)
- Loại bỏ các phép toán dư thừa
- Tận dụng Caching

Hãy giải thích các cải tiến và hiệu quả dự kiến.
```

---

## 3. Cập nhật mã nguồn Hiện đại (Modern Syntax)

```
Cập nhật đoạn code này sang cú pháp [JavaScript/TypeScript] hiện đại:

[Dán code cũ ở đây]

Sử dụng:
- Các tính năng ES6+ (arrow functions, destructuring, spread, v.v.)
- async/await thay cho callbacks/promises cũ
- Optional chaining và nullish coalescing
- Các phương thức mảng hiện đại (map, filter, reduce)

Yêu cầu: Đảm bảo khả năng tương thích với môi trường [tên môi trường, ví dụ: Node 20].
```

---

## 4. Chuyển đổi Thư viện (Migration)

```
Hãy giúp tôi chuyển đổi code từ [thư viện cũ] sang [thư viện mới]:

Đoạn code hiện tại dùng [thư viện cũ]:
[Dán code ở đây]

Yêu cầu:
- Giữ nguyên các chức năng cũ
- Phiên bản thư viện mới: [phiên bản]
- Có hỗ trợ TypeScript không: [Có/Không]

Hãy cung cấp:
1. Các gói cần cài đặt/thay đổi.
2. Code sau khi chuyển đổi.
3. Các lưu ý quan trọng về thay đổi lớn (breaking changes).
```
